Remote file systems have more failure modes. If a client uses a remote file system, it perform file open operation from remote host, look up to open files, reading or writing data to files, and closing files. Now if partition of network is performed, or a crash of the server, or scheduled shutdown of server, suddenly, remote file system either terminates all operation to lost server on delay operations until the server is again reachable.
