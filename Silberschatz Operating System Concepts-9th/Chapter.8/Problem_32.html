Step-1

a.

Translating the logical address into physical address:

An Intel Pentium process CPU generates a logical address. which involves following phases to convert it into physical addresses:

1. Segmentation unit

2. Paging unit

Step-2

Segmentation unit:

• The logical address is given to the segmentation unit.

• Logical address has two fields which are selector and offset.

o Selector is a 16-bit field which is used as an index to the descriptor table.

o Each segment has three fields which are segment number (s), whether the segment is a Local or Global Descriptor table (LDT or GDT) and protection field (p).

• The selector filed selects one of the segments and perform an EX-OR with the offset to generate a 32-bit linear address.

Step-3

Paging unit:

• The 32-bit linear address is given to the paging unit. It has three fields which are dir, page and the offset.

• The filed dir is used as an index to the page directory. The entry from the page directory selects the page table.

• The filed page is used as an index to the page table.

• The entry of the page table is EX-ORed with the offset to generate the physical address.

• The physical memory is accessed by the physical address.

Step-4

b. Advantages to the address-translation system:

• The page translation mechanism of an Intel processor provided more flexibility and the kernel implementation is simple.

• An operating system implements the entire memory scheme in hardware, instead of having to implement some parts in hardware and some parts in software.

• Therefore, the efficiency of the Intel Pentium processor is increasing.

Step-5

c. Disadvantages to the address-translation system:

• The translation of logical address into physical address involves multiple table lookups and it takes longer due to invoke each table.

• Cache is used in the segmentation unit to read the descriptor but, there will still be cache misses.
